# 11. 国际化 (i18n) 系统

> **写给零基础的你**：国际化（i18n）就是让你的软件能说"多国语言"。比如你的软件在中国用显示"确定"，在美国用显示"OK"，在日本用显示"確認"。i18n 这个名字来自 "internationalization" 这个单词——i 和 n 之间有 18 个字母，所以简写为 i18n。

## 11.1 概述

国际化（i18n）是将应用适配多种语言和地区的过程。学完本章，你将能够：

- 理解 Avalonia 应用的国际化架构设计
- 实现基于 JSON 资源文件的翻译系统
- 创建自定义 `MarkupExtension` 在 AXAML 中声明式使用翻译
- 理解 `IObservable` 模式与 Avalonia 绑定系统的集成
- 处理复数形式、日期/数字本地化格式
- 支持 RTL（从右到左）语言
- 使用源生成 JSON 序列化确保 AOT 兼容
- 建立翻译工作流和质量保证流程

CodexSwitch 实现了一套完整的国际化系统，支持中文、英文、日文等多种语言，所有用户可见文本都通过 i18n 系统管理，切换语言后 UI 立即更新。

## 11.2 核心概念

### 11.2.1 架构概览

> **小白提示**：国际化系统就像一个"翻译团队"：
> - **翻译文件**（JSON）= 各语言的翻译词典
> - **I18nService** = 翻译经理，负责查找和分发翻译
> - **TrExtension** = 翻译机器人，自动在界面上显示正确的语言
> - **IObservable** = 当语言切换时，自动通知所有界面更新

```
Assets/i18n/
├── languages.json       ← 语言目录（定义支持的语言列表，就像菜单）
├── zh-CN.json           ← 中文翻译词典
├── en-US.json           ← 英文翻译词典
└── ja-JP.json           ← 日文翻译词典

I18n/
├── I18nCatalog.cs       ← 数据模型（翻译词典的结构）
├── I18nService.cs       ← 翻译服务（单例，运行时语言切换）
└── TrExtension.cs       ← AXAML MarkupExtension（声明式绑定翻译）
```

### 11.2.2 数据流

```
avares:// 资源文件
    -> I18nService.LoadDefault() 解析 JSON
    -> I18nService.Current 单例持有翻译表
    -> TrExtension 提供 IObservable<string>
    -> Avalonia 绑定系统自动更新 UI
```

### 11.2.4 I18nService 核心实现

**加载翻译资源**

```csharp
public sealed class I18nService : INotifyPropertyChanged
{
    public const string CatalogResourceUri = "avares://CodexSwitch/Assets/i18n/languages.json";

    public static I18nService Current { get; } = LoadDefault();

    private readonly Dictionary<string, Dictionary<string, string>> _translations;

    public static I18nService LoadDefault()
    {
        var catalog = LoadCatalog();
        var resources = catalog.Languages
            .Select(language => LoadLanguageResource(language.Code))
            .ToArray();
        return new I18nService(catalog, resources);
    }

    public static I18nCatalog LoadCatalog()
    {
        using var stream = AssetLoader.Open(new Uri(CatalogResourceUri));
        return DeserializeCatalog(stream);
    }

    public static I18nLanguageResource LoadLanguageResource(string code)
    {
        var resourceUri = $"avares://CodexSwitch/Assets/i18n/{code}.json";
        using var stream = AssetLoader.Open(new Uri(resourceUri));
        return DeserializeLanguageResource(stream, code);
    }
}
```

`avares://` 是 Avalonia 的嵌入资源协议。编译时，`Assets/i18n/*.json` 被嵌入到程序集中，运行时通过 `AssetLoader.Open` 读取。

**翻译查找与回退**

```csharp
public string Translate(string key)
{
    if (string.IsNullOrWhiteSpace(key))
        return "";

    // 先查当前语言
    if (TryTranslate(CurrentLanguageCode, key, out var current))
        return current;

    // 回退到默认语言
    return TryTranslate(DefaultLanguageCode, key, out var fallback)
        ? fallback
        : key;  // 最终回退：返回 key 本身
}

private bool TryTranslate(string languageCode, string key, out string value)
{
    value = "";
    return _translations.TryGetValue(languageCode, out var table) &&
        table.TryGetValue(key, out value!);
}
```

回退策略：当前语言 -> 默认语言 -> 返回 key。这确保了即使翻译缺失，UI 也不会显示空白。

**格式化翻译**

```csharp
public string Format(string key, params object?[] args)
{
    return string.Format(CultureInfo.CurrentCulture, Translate(key), args);
}
```

使用方式：

```csharp
var message = I18nService.Current.Format("provider.delete.confirm", providerName);
// zh-CN: "确定要删除 OpenAI 吗？"
// en-US: "Are you sure you want to delete OpenAI?"
```

**语言切换**

```csharp
public event EventHandler? LanguageChanged;

public bool SetLanguage(string? code)
{
    var resolved = ResolveLanguageCode(code);
    if (string.Equals(CurrentLanguageCode, resolved, StringComparison.OrdinalIgnoreCase))
        return false;

    CurrentLanguageCode = resolved;
    PropertyChanged?.Invoke(this, new PropertyChangedEventArgs("Item[]"));
    PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(nameof(CurrentLanguageCode)));
    PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(nameof(CurrentLanguage)));
    LanguageChanged?.Invoke(this, EventArgs.Empty);
    return true;
}
```

`SetLanguage` 触发两个通知通道：
- `PropertyChanged`：通知 Avalonia 绑定系统（索引器语法）
- `LanguageChanged`：通知 `TrExtension` 的 `IObservable` 订阅者

## 11.3 进阶用法

### 11.3.1 TrExtension — 自定义 MarkupExtension

`TrExtension` 是在 AXAML 中使用翻译的核心。完整实现：

```csharp
public sealed class TrExtension : MarkupExtension
{
    public TrExtension() { }

    public TrExtension(string key)
    {
        Key = key;
    }

    public string Key { get; set; } = "";

    public override object ProvideValue(IServiceProvider serviceProvider)
    {
        return new TranslationObservable(Key).ToBinding();
    }

    private sealed class TranslationObservable(string key) : IObservable<string>
    {
        public IDisposable Subscribe(IObserver<string> observer)
        {
            // 立即推送当前翻译值
            observer.OnNext(I18nService.Current.Translate(key));

            // 订阅语言变更事件
            EventHandler handler = (_, _) =>
                observer.OnNext(I18nService.Current.Translate(key));
            I18nService.Current.LanguageChanged += handler;

            // 返回取消订阅的 IDisposable
            return new ActionDisposable(() =>
                I18nService.Current.LanguageChanged -= handler);
        }
    }

    private sealed class ActionDisposable(Action dispose) : IDisposable
    {
        private Action? _dispose = dispose;

        public void Dispose()
        {
            Interlocked.Exchange(ref _dispose, null)?.Invoke();
        }
    }
}
```

**工作原理**

1. `ProvideValue` 被 AXAML 编译器调用，返回一个 `IBinding`
2. 内部的 `TranslationObservable` 实现 `IObservable<string>`
3. 当 Avalonia 绑定系统订阅时，`Subscribe` 立即推送当前翻译
4. `LanguageChanged` 事件触发时，再次推送新翻译值
5. `ActionDisposable` 使用 `Interlocked.Exchange` 确保 `Dispose` 只执行一次

**在 AXAML 中使用**

```xml
<!-- 命名空间声明 -->
xmlns:i18n="using:CodexSwitch.I18n"

<!-- 基本用法 -->
<TextBlock Text="{i18n:Tr app.name}"/>
<TextBlock Text="{i18n:Tr nav.home}"/>
<Window Title="{i18n:Tr app.name}"/>
<Button Content="{i18n:Tr provider.add}"/>

<!-- 也可以用构造函数参数语法 -->
<TextBlock Text="{i18n:Tr Key=nav.settings}"/>
```

### 11.3.2 IObservable 模式的优势

传统的 i18n 实现通常使用 `INotifyPropertyChanged` 和索引器绑定：

```xml
<!-- 传统方式 -->
<TextBlock Text="{Binding [nav.home], Source={x:Static i18n:I18nService.Current}}"/>
```

CodexSwitch 的 `TrExtension` 使用 `IObservable<string>` + `ToBinding()`：

```xml
<!-- CodexSwitch 方式 -->
<TextBlock Text="{i18n:Tr nav.home}"/>
```

| 特性 | 传统索引器 | TrExtension |
|------|-----------|-------------|
| 语法 | `{Binding [key]}` | `{i18n:Tr key}` |
| Source 声明 | 需要 | 不需要 |
| 独立订阅 | 共享 PropertyChanged | 每个绑定独立订阅 |
| 生命周期管理 | 依赖绑定系统 | `ActionDisposable` 精确管理 |
| AOT 兼容 | 需要特殊处理 | 天然兼容 |

### 11.3.3 构造函数与字典初始化

```csharp
public I18nService(I18nCatalog catalog, IEnumerable<I18nLanguageResource> resources)
{
    _languages = catalog.Languages
        .Where(l => !string.IsNullOrWhiteSpace(l.Code))
        .GroupBy(l => l.Code.Trim(), StringComparer.OrdinalIgnoreCase)
        .ToDictionary(g => g.Key, g => g.First(), StringComparer.OrdinalIgnoreCase);

    _translations = resources
        .Where(r => !string.IsNullOrWhiteSpace(r.Code))
        .GroupBy(r => r.Code.Trim(), StringComparer.OrdinalIgnoreCase)
        .ToDictionary(
            g => group.Key,
            g => new Dictionary<string, string>(g.First().Translations, StringComparer.Ordinal),
            StringComparer.OrdinalIgnoreCase);
}
```

关键细节：
- `StringComparer.OrdinalIgnoreCase` 使语言 code 查找不区分大小写
- `StringComparer.Ordinal` 使翻译 key 区分大小写——`nav.Home` 和 `nav.home` 是不同的 key
- `GroupBy` + `ToDictionary` 去重，防止 JSON 中有重复的 language code

### 11.3.4 源生成 JSON 序列化

为了 Native AOT 兼容，CodexSwitch 使用源生成的 JSON 序列化：

```csharp
[JsonSerializable(typeof(I18nCatalog))]
[JsonSerializable(typeof(I18nLanguageResource))]
internal partial class CodexSwitchJsonContext : JsonSerializerContext { }

// 使用时传入源生成的上下文
public static I18nCatalog DeserializeCatalog(Stream stream)
{
    return JsonSerializer.Deserialize(stream, CodexSwitchJsonContext.Default.I18nCatalog)
        ?? new I18nCatalog();
}
```

`JsonSerializerContext` 在编译时生成序列化代码，避免运行时反射，是 AOT 发布的必要条件。

对比：

```csharp
// AOT 不兼容：运行时反射
JsonSerializer.Deserialize<I18nCatalog>(stream);

// AOT 兼容：源生成
JsonSerializer.Deserialize(stream, CodexSwitchJsonContext.Default.I18nCatalog);
```

## 11.4 组件详解大全

### 11.4.1 .resx 资源文件 vs JSON 资源文件

.NET 传统方式使用 `.resx` 资源文件，CodexSwitch 选择 JSON 资源文件。

| 特性 | .resx | JSON |
|------|-------|------|
| 编辑工具 | Visual Studio 内置 | 任何文本编辑器 |
| 格式 | XML | JSON |

### 11.4.2 复数形式处理

不同语言的复数规则不同。英文有单复数，中文无复数，阿拉伯语有多种复数形式。

```csharp
public static class PluralHelper
{
    public static string TranslatePlural(string key, int count)
    {
        var suffix = GetPluralSuffix(count);
        var pluralKey = $"{key}.{suffix}";
        var result = I18nService.Current.Translate(pluralKey);

        // 如果没有找到复数形式，回退到默认形式
        if (result == pluralKey)
            result = I18nService.Current.Translate(key);

        return string.Format(result, count);
    }

    private static string GetPluralSuffix(int count)
    {
        // 简化的英文复数规则
        return count switch
        {
            0 => "zero",
            1 => "one",
            _ => "other"
        };
    }
}
```

翻译文件中的复数形式：

```json
{
  "item.count.zero": "No items",
  "item.count.one": "1 item",
  "item.count.other": "{0} items"
}
```

```json
{
  "item.count.zero": "没有项目",
  "item.count.one": "1 个项目",
  "item.count.other": "{0} 个项目"
}
```

使用方式：

```csharp
var text = PluralHelper.TranslatePlural("item.count", 5);
// en-US: "5 items"
// zh-CN: "5 个项目"
```

### 11.4.3 日期/数字的本地化格式

```csharp
// 数字格式化
```


### 11.4.4 RTL 语言支持（FlowDirection）

对于阿拉伯语、希伯来语等从右到左的语言：

```xml
<!-- 在 Window 或 UserControl 上设置 -->
<Window FlowDirection="{Binding FlowDirection}">

<!-- 或根据语言自动设置 -->
<Window FlowDirection="{i18n:Tr flow.direction}">
```

```csharp
public static FlowDirection GetFlowDirection(string languageCode)
{
    return languageCode switch
    {
        "ar-SA" => FlowDirection.RightToLeft,
        "he-IL" => FlowDirection.RightToLeft,
        _ => FlowDirection.LeftToRight
    };
}
```

**RTL 布局注意事项**

```xml
<!-- 好：使用 Start/End 代替 Left/Right -->
<Button HorizontalAlignment="Start"/>  <!-- RTL 时自动变成 Right -->

<!-- 差：硬编码 Left/Right -->
<Button HorizontalAlignment="Left"/>   <!-- RTL 时仍是 Left，布局错误 -->
```

| LTR 属性 | RTL 等价 | 建议 |
|----------|---------|------|
| `Left` | `Right` | 使用 `Start`/`End` |
| `Right` | `Left` | 使用 `Start`/`End` |
| `Margin="10,0,0,0"` | `Margin="0,0,10,0"` | 使用 `FlowDirection` 感知 |
| `Padding="10,20,30,40"` | `Padding="30,20,10,40"` | 使用 `FlowDirection` 感知 |

3. **翻译覆盖率检查**

```csharp
[Fact]
public void AllTranslationKeysArePresent()
{
    var catalog = I18nService.LoadCatalog();
    var defaultResource = I18nService.LoadLanguageResource(catalog.DefaultLanguage);
    var defaultKeys = defaultResource.Translations.Keys.ToHashSet();

    foreach (var language in catalog.Languages)
    {
        if (language.Code == catalog.DefaultLanguage) continue;

        var resource = I18nService.LoadLanguageResource(language.Code);
        var keys = resource.Translations.Keys.ToHashSet();

        var missing = defaultKeys.Except(keys).ToList();
        Assert.True(missing.Count == 0,
            $"Language {language.Code} is missing keys: {string.Join(", ", missing)}");
    }
}
```

4. **AXAML 翻译 key 提取**

```csharp
[Fact]
public void AllAxamlTranslationKeysAreTranslated()
{
    // 扫描所有 AXAML 文件中的 {i18n:Tr ...} 用法
    var axamlFiles = Directory.GetFiles(projectPath, "*.axaml", SearchOption.AllDirectories);
    var usedKeys = new HashSet<string>();

    foreach (var file in axamlFiles)
    {
        var content = File.ReadAllText(file);
        var matches = Regex.Matches(content, @"\{i18n:Tr\s+(\S+?)\}");
        foreach (Match match in matches)
            usedKeys.Add(match.Groups[1].Value);
    }

    var catalog = I18nService.LoadCatalog();
    var defaultResource = I18nService.LoadLanguageResource(catalog.DefaultLanguage);

    foreach (var key in usedKeys)
    {
        Assert.True(defaultResource.Translations.ContainsKey(key),
            $"Translation key '{key}' used in AXAML but not defined in translations");
    }
}
```

### 11.5.2 语言自动检测

```csharp
public static string DetectLanguage(IReadOnlyList<I18nLanguageOption> languages)
{
    var culture = CultureInfo.CurrentCulture.Name;  // 例如 "zh-CN"
    if (languages.Any(l => string.Equals(l.Code, culture, StringComparison.OrdinalIgnoreCase)))
        return culture;

    // 尝试匹配语言前缀
    var prefix = culture.Split('-')[0];
    var match = languages.FirstOrDefault(l =>
        l.Code.StartsWith(prefix, StringComparison.OrdinalIgnoreCase));
    return match?.Code ?? languages.FirstOrDefault()?.Code ?? "en-US";
}
```

## Deep Dive：内部原理

### IObservable 与绑定系统的集成

`TranslationObservable.Subscribe` 返回的 `IDisposable` 会被 Avalonia 绑定系统持有。当控件从视觉树移除时，绑定系统调用 `Dispose`，取消对 `LanguageChanged` 事件的订阅。这避免了已销毁控件的内存泄漏。

```
控件添加到视觉树
    -> 绑定系统调用 Subscribe()
    -> TranslationObservable 推送当前翻译
    -> UI 显示翻译文本

语言切换
    -> I18nService.SetLanguage()
    -> 触发 LanguageChanged 事件
    -> TranslationObservable 推送新翻译
    -> UI 更新

控件从视觉树移除
    -> 绑定系统调用 Dispose()
    -> ActionDisposable 取消订阅
    -> 不再接收 LanguageChanged 事件
```

### 线程安全

`I18nService.Current` 是静态只读单例，在类加载时初始化。`_translations` 字典在构造后不再修改，因此 `Translate` 方法天然线程安全。`SetLanguage` 通过事件通知 UI 线程更新。

### avares:// 资源加载

```
编译时：
    Assets/i18n/*.json -> 嵌入到程序集

运行时：
    AssetLoader.Open(new Uri("avares://CodexSwitch/Assets/i18n/zh-CN.json"))
    -> 从嵌入资源中读取 Stream
    -> JsonSerializer.Deserialize() 解析
```

## Common Pitfalls

### 陷阱 1：忘记同步更新所有语言文件

添加新 key 时，必须在所有 `*.json` 翻译文件中添加对应条目。否则缺失的语言会回退到默认语言或显示 key 本身。

### 陷阱 2：翻译 key 大小写不匹配

```csharp
// AXAML 中使用 app-name（连字符）
<TextBlock Text="{i18n:Tr app-name}"/>

// JSON 中使用 app.name（点号）
{ "app.name": "CodexSwitch" }
```

`I18nService` 使用 `StringComparer.Ordinal` 比较 key，大小写和标点必须完全匹配。

### 陷阱 3：未使用源生成序列化导致 AOT 失败

```csharp
// 错误：运行时反射序列化
JsonSerializer.Deserialize<I18nCatalog>(stream);

// 正确：使用源生成上下文
JsonSerializer.Deserialize(stream, CodexSwitchJsonContext.Default.I18nCatalog);
```

### 陷阱 4：TranslationObservable 订阅泄漏

如果 `Subscribe` 返回的 `IDisposable` 未被正确调用，`LanguageChanged` 事件会持有已销毁控件的引用。`ActionDisposable` 使用 `Interlocked.Exchange` 确保 `Dispose` 只执行一次。

### 陷阱 5：在非 UI 线程调用 SetLanguage

`SetLanguage` 触发的事件会被 Avalonia 绑定系统在 UI 线程处理。如果从后台线程调用，可能导致跨线程异常。应通过 `Dispatcher.UIThread.Post` 切换到 UI 线程。

### 陷阱 6：翻译文件编码问题

确保所有 JSON 文件使用 UTF-8 编码（无 BOM）。某些编辑器会添加 BOM，导致 JSON 解析失败。

### 陷阱 7：格式化字符串中的转义

```json
{
  "percentage": "完成率: {0:P0}",
  "dollar": "价格: ${0:F2}"
}
```

`{0}` 是 `string.Format` 的占位符。如果翻译文本中需要显示 `{` 或 `}`，使用 `{{` 和 `}}` 转义。

### 陷阱 8：StaticResource vs DynamicResource 与 i18n

翻译文本通过 `TrExtension` 返回 `IBinding`，这与 `DynamicResource` 类似——语言切换时自动更新。不要在翻译文本上使用 `StaticResource`，否则不会响应语言切换。

### 陷阱 9：翻译文件中嵌套 JSON 结构

翻译文件只支持扁平的 key-value 结构，不支持嵌套对象。使用点号分隔的命名空间来组织 key。

### 陷阱 10：翻译 key 与代码中的硬编码字符串

```csharp
// 错误：代码中硬编码字符串
MessageBox.Show("确定要删除吗？");

// 正确：使用翻译服务
MessageBox.Show(I18nService.Current.Translate("provider.delete.confirm"));
```

### 陷阱 11：忽略 ICU 消息格式差异

不同语言的句子结构不同。例如：

```json
{
  "result": "找到 {0} 个结果，共 {1} 页"
}
```

日语可能是：`"{1} ページ中 {0} 件の結果が見つかりました"`

参数顺序可能不同，使用命名参数可以提高可读性：

```csharp
string.Format(template, results, pages)
```
