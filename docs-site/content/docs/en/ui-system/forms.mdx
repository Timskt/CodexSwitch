Form components cover the main provider, model, settings, and auth workflows.

## Component families

- Button
- Button Group for related action clusters, connected item anatomy, separators, text, nested groups, and input composition
- Split Button for primary commands plus Web-style primary pointer release and Enter/Space/ArrowDown menu trigger state, source-aware `OpenChanged`, close-on-select action rows, loading suppression, and focus return
- Input Group for root border/focus ring anatomy, inline and block addons, action buttons, keyboard hints, and textarea footers
- Input OTP for grouped one-time password slots, slot anatomy, separators, paste entry, pattern filtering, invalid state, and keyboard navigation
- Label for target-associated form text, required-marker anatomy, access keys, intent styling, and disabled target styling
- Field labels, descriptions, required state, messages, horizontal/responsive orientation, FieldGroup, FieldSet, FieldContent, FieldSeparator, and FieldError
- TextBox
- Select for trigger, selected value, placeholder, popup motion, disabled items, source-aware `ValueChanged` parity with Web `onValueChange`, and source-aware `OpenChanged` parity with Web `onOpenChange`
- Combobox for filterable predefined selection, primary pointer release trigger parity, source-aware open and closed popup motion, highlighted and selected items, `SelectionChanged` old/new item/index/value/source metadata, clear action, loading and empty slots, and Arrow, Enter, and Escape behavior
- Native Select for option values, optgroup labels, disabled options, invalid state, native selection behavior, source-aware `ValueChanged` old/new option values, and source-aware `OpenChanged` popup state
- Calendar for single dates, ranges, outside days, week numbers, disabled bounds, command-blocked day buttons, keyboard navigation, and source-aware `SelectedDateChanged`, `RangeChanged`, `DisplayDateChanged`, and `ActiveDateChanged` events
- Date Picker for popover plus calendar composition, primary pointer release trigger parity, single and range selection, clear action, open/close motion, Escape dismissal, loading/disabled guards, and source-aware open/select/range events
- Checkbox for label-target anatomy, checked/unchecked/indeterminate state classes, source-aware `CheckedStateChanged` parity with Web `onCheckedChange`, disabled guards, intent, and Field/FieldSet composition
- Radio and Radio Group for root-owned value, item values, primary pointer release item activation, `ValueChanged` old/new item/index/value/source metadata, orientation, loop, required, disabled, and loading suppression
- Switch and standalone Toggle for two-state controls with source-aware `CheckedChanged` / `PressedChanged` Web event parity, plus Toggle Group as its own page for single/multiple values, source-aware `ValueChanged`, `spacing=2` default gaps, `spacing=0` connected items, vertical orientation, disabled items, and roving keyboard focus
- Slider for tokenized track/thumb states, hover/pressed/dragging motion, `ValueChanging` parity with Web `onValueChange`, and `ValueCommitted` parity with Web `onValueCommit`

Field coverage now follows the current shadcn Field contract: grouped fieldsets, legends, responsive field groups, content/title/description slots, inline separators, invalid state, and one-or-many error messages all render as component-owned primitives with inline AXAML examples in Docs. Button Group, Input Group, Input OTP, and Label now each have separate anatomy cases so connected item shape, addon alignment, grouped slots, separators, target association, required markers, and disabled-target styling are not hidden inside composition samples. Select coverage now includes default, state, anatomy, and interaction cases, with each rendered example exposing its own local AXAML code expansion below the component plus source metadata for pointer, keyboard, and programmatic value/open changes. Native Select now has separate state, anatomy, composition, and interaction cases: anatomy breaks out the trigger, selected content, chevron, popup surface, option values, optgroup labels, disabled options, invalid compact trigger, and local AXAML expansion while preserving the same explicit source-aware value/open event path. Combobox interaction coverage now surfaces source-aware OpenChanged for pointer, keyboard, input, focus, clear, item, and programmatic paths alongside selection metadata. Calendar now has separate state, anatomy, composition, and interaction cases: anatomy breaks out the header, weekday grid, outside days, range edges, week numbers, and bounded day cells, while interaction surfaces DayPicker-style `onSelect` and `onMonthChange` parity through selected-date, range, active-date, display-date, and command-blocked day activation paths with pointer/keyboard/programmatic source metadata. Date Picker forwards the same source metadata through popover open/close, Calendar selection sync, range completion, and keyboard clear paths. Switch, Checkbox, and standalone Toggle interaction cases now surface live checked/pressed changes with pointer/keyboard/programmatic source metadata while keeping the local AXAML expansion under each rendered component. Toggle Group coverage is also split from standalone Toggle so every group state, anatomy, and interaction case has its own rendered example and local AXAML code expansion, including pointer/keyboard/programmatic value-change source metadata. Slider coverage now includes value-change and value-commit event previews, keyboard commit behavior, vertical orientation, disabled guards, and per-example AXAML expansion.
