Feedback components make background operations legible: provider state, proxy state, OAuth progress, update checks, empty results, loading, and usage refreshes.

## Review states

- Success, warning, error, and neutral variants.
- Alert anatomy: icon, title, description, rich content, action slot, and default glyph behavior.
- Sonner service lifecycle: show, dismiss, clear, visible stack, toast limit trimming, loading duration, close-button visibility, and rich-color viewport states.
- Badge anatomy and interaction: variants, status dots, inline icon content, spinner content, link-like badges, command activation, focus-visible, keyboard activation, and `Activated` source metadata.
- Avatar and AvatarGroup anatomy and interaction: image loading status, delayed fallback, error fallback, status badge, overlapping stacks, inherited sizing, hidden members, and AvatarGroupCount overflow members.
- Long text wrapping.
- Loading and empty states.
- Empty-state primary and secondary actions, including loading, disabled, and command-blocked trigger suppression with visible action state classes.
- Reduced-motion expectations.
