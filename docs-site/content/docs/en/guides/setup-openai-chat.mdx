Use `OpenAiChat` when an upstream service exposes `/chat/completions` but the local client still sends Responses-style requests to CodexSwitch.

## Runtime conversion

CodexSwitch converts inbound Responses requests into Chat Completions payloads, forwards them upstream, then converts the result back to Responses shape.

## Quick setup

1. Create or choose a Chat-compatible provider.
2. Set protocol to `OpenAiChat`.
3. Enter the base URL and API key.
4. Map local model names to upstream chat model names.
5. Send a simple response request before testing tool-heavy prompts.

## Troubleshooting

If streaming shape or tool calls look wrong, compare the upstream service's Chat Completions behavior with the adapter expectations in [Protocols](/en/docs/core-concepts/protocols).
