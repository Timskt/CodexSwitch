Use `OpenAiResponses` when the upstream service accepts Responses-style requests directly.

## Quick setup

1. Add a provider from a built-in OpenAI Responses template or create a custom provider.
2. Set protocol to `OpenAiResponses`.
3. Enter the upstream base URL and API key.
4. Configure default model and any upstream model remapping.
5. Set the provider as active.

## Verify

```text
GET http://127.0.0.1:12785/v1/models
POST http://127.0.0.1:12785/v1/responses
```

If `/v1/models` works but `/v1/responses` fails, check model remapping, service tier overrides, and upstream auth.
