Network behavior combines local binding, outbound provider access, provider availability, and model fallback rules.

## Defaults

- Keep the local proxy host on `127.0.0.1`.
- Use provider-specific base URLs for upstream traffic.
- Confirm `/health` before debugging upstream providers.
- Confirm `/v1/models` before debugging request conversion.

## Failover checks

1. Confirm the requested model is supported by the active provider.
2. Confirm a fallback provider supports the same model or alias.
3. Confirm disabled providers are not expected to receive traffic.
4. Check provider auth before assuming routing is wrong.
