The `ci` workflow validates changelog state, restores packages, builds the test project, runs xUnit tests, and publishes Native AOT artifacts for platform runners.

## Standard checks

```powershell
dotnet restore CodexSwitch.Tests/CodexSwitch.Tests.csproj
dotnet build CodexSwitch.Tests/CodexSwitch.Tests.csproj -c Release --no-restore
dotnet test CodexSwitch.Tests/CodexSwitch.Tests.csproj -c Release --no-build --no-restore
```

## Release trigger

When the workflow runs on a `vX.Y.Z` tag, it checks that the tag matches `Directory.Build.props`, creates the GitHub Release, and uploads generated artifacts.
