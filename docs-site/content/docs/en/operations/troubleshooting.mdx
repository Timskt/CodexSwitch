Start with the local proxy, then provider visibility, then request conversion. This keeps auth and routing problems from looking like protocol bugs.

## Proxy ladder

```powershell
curl http://127.0.0.1:12785/health
curl http://127.0.0.1:12785/v1/models
```

Good output:

```text
/health returns 200 OK
/v1/models returns model data for the active provider
```

## Provider ladder

1. Confirm the provider is active.
2. Confirm the API key or OAuth account is present.
3. Confirm the requested model is mapped to an upstream model.
4. Confirm the provider protocol matches the upstream service.
5. Confirm a disabled provider is not expected to handle traffic.

## Managed file ladder

1. Check whether `.codex` or `.claude` managed files were written.
2. Check for same-name `.bak` backups.
3. Stop the proxy and verify original files restore.
4. If using Codex App plugins, confirm auth preservation is enabled before applying proxy config.

## Release ladder

- Run `pwsh ./build/Validate-Changelog.ps1`.
- Confirm the tag is `vX.Y.Z`.
- Confirm the version in `Directory.Build.props` matches the changelog heading.
- For macOS, check signing/notarization secrets if Gatekeeper blocks downloads.
