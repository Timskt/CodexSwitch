CodexSwitch combines a desktop control plane with an in-process local proxy. The UI owns provider configuration, model routing, usage, settings, OAuth, and updates. The proxy exposes local OpenAI Responses-style routes and forwards each request to the selected upstream provider.

```text
Codex client -> CodexSwitch local /v1 proxy -> Provider routing
  -> OpenAI Responses
  -> OpenAI-compatible Chat
  -> Anthropic Messages
  -> Usage and cost log
```

Read these concepts before changing routing, protocol conversion, managed config, or usage accounting.
