---
title: Providers
description: Built-in and custom upstream services that CodexSwitch can route to.
icon: Boxes
---

A provider describes an upstream service, its protocol, credentials, model routes, model names, service tiers, and request overrides.

## Built-in templates

| Provider | Default protocol | Example default model |
| --- | --- | --- |
| RoutinAI | OpenAI Responses | `gpt-5.5` |
| RoutinAI Plan | OpenAI Responses | `gpt-5.5` |
| OpenAI Official | OpenAI Responses | `gpt-5.5` |
| Anthropic Messages | Anthropic Messages | `claude-sonnet-4-5` |
| DeepSeek | OpenAI Chat | `deepseek-v4-flash` |
| Xiaomi MiMo | OpenAI Chat | `mimo-v2.5-pro` |
| Codex OAuth | OpenAI Responses | `gpt-5.1-codex` |

## Custom providers

Use a custom provider when an upstream service is compatible with OpenAI Responses, OpenAI Chat Completions, or Anthropic Messages. Configure the base URL, API key, protocol, model routes, upstream model names, and service tiers.
