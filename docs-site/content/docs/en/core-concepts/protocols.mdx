CodexSwitch exposes Responses-style inbound routes and adapts outbound requests to the provider protocol.

| Provider protocol | Upstream endpoint | Notes |
| --- | --- | --- |
| `OpenAiResponses` | `/responses` | Passes Responses requests through with model, service-tier, and request override handling. |
| `OpenAiChat` | `/chat/completions` | Converts inbound Responses requests to Chat Completions, then converts results back to Responses shape. |
| `AnthropicMessages` | `/messages` | Converts inbound Responses or Messages traffic to Anthropic-compatible payloads where supported. |

CodexSwitch does not expose `/v1/chat/completions` as a public inbound API.
