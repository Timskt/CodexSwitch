Model routing starts with the requested model and the active provider for the calling client. `ProviderRoutingResolver` chooses a provider, applies aliases and upstream remapping, and can fall through to another provider that supports the requested model.

## Routing inputs

- Requested model name.
- Active provider for Codex or Claude Code.
- Provider protocol.
- Model aliases and upstream model names.
- Service tier and request override settings.
- Provider availability and disabled state.

## Verification

Use `/v1/models` to confirm what the local proxy exposes before testing `/v1/responses`.
