Provider and protocol changes usually touch more than one layer.

## Checklist

- Add or update provider template data.
- Confirm protocol selection and model remapping.
- Update adapter behavior only within the relevant protocol boundary.
- Normalize usage so dashboard and pricing stay correct.
- Preserve managed config restore behavior.
- Add tests for routing, conversion, usage, and config changes.

Use [Protocols](/en/docs/core-concepts/protocols) and [Models And Routes](/en/docs/core-concepts/models-and-routes) as the mental model before editing.
