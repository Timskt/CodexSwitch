Codex talks to CodexSwitch as if it were an OpenAI Responses endpoint. CodexSwitch then chooses the active provider, applies routing rules, converts protocols when needed, and records usage locally.

## Quick setup

1. Open CodexSwitch.
2. Choose a built-in provider template or create a custom provider.
3. Add the provider API key, or use the Codex OAuth login button for the Codex OAuth provider.
4. Set the provider as active.
5. Keep the local proxy enabled.

Codex clients can use:

```text
http://127.0.0.1:12785/v1
```

## Good output looks like

```text
GET /health -> 200 OK
GET /v1/models -> returns the active provider model list
POST /v1/responses -> streams or returns a Responses-style result
```

## What changes locally

When the proxy starts, CodexSwitch writes managed `.codex` and `.claude` files and keeps originals as same-name `.bak` backups. When the proxy stops, originals are restored from those backups.
