模型路由从请求中的模型名和当前客户端的活动 provider 开始。`ProviderRoutingResolver` 会选择 provider，应用别名和上游模型映射，并在可行时回退到支持该模型的其他 provider。

## 路由输入

- 请求模型名。
- Codex 或 Claude Code 的活动 provider。
- Provider 协议。
- 模型别名和上游模型名。
- Service tier 和请求覆盖设置。
- Provider 可用性和禁用状态。

## 验证

先用 `/v1/models` 确认本地代理暴露了哪些模型，再测试 `/v1/responses`。
