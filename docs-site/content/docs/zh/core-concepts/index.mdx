CodexSwitch 由桌面控制面和进程内本地代理组成。UI 负责 provider 配置、模型路由、用量、设置、OAuth 和更新；代理暴露本地 OpenAI Responses 风格端点，并把每个请求转发到选中的上游 provider。

```text
Codex client -> CodexSwitch local /v1 proxy -> Provider routing
  -> OpenAI Responses
  -> OpenAI-compatible Chat
  -> Anthropic Messages
  -> Usage and cost log
```

修改路由、协议转换、受管理配置或用量统计前，先阅读本分区。
