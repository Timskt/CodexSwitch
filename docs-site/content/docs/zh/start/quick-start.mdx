Codex 会把 CodexSwitch 当成一个 OpenAI Responses 端点。CodexSwitch 收到请求后选择活动 provider，应用路由规则，在需要时转换协议，并把用量写入本地日志。

## 快速配置

1. 打开 CodexSwitch。
2. 选择一个内置 provider 模板，或创建自定义 provider。
3. 填入 API key；如果使用 Codex OAuth provider，则点击 Codex OAuth 登录。
4. 将 provider 设置为当前活动 provider。
5. 保持本地代理启用。

Codex 客户端使用：

```text
http://127.0.0.1:12785/v1
```

## 正常输出

```text
GET /health -> 200 OK
GET /v1/models -> 返回活动 provider 的模型列表
POST /v1/responses -> 返回或流式输出 Responses 风格结果
```

## 本地文件变化

代理启动时，CodexSwitch 会写入受管理的 `.codex` 和 `.claude` 文件，并把原文件保留为同名 `.bak` 备份。代理停止时，会从备份恢复原文件。
