CodexSwitch 使用仓库内单一版本来源，并通过 tag 触发的 CI 自动创建 GitHub Release。

## 版本来源

- 更新 `Directory.Build.props` 中的 `Version`、`AssemblyVersion`、`FileVersion` 和 `InformationalVersion`。
- Git tag 和 GitHub Release tag 保持 `vX.Y.Z` 格式。
- `CHANGELOG.md` 中第一个发布小节要与当前版本一致，例如 `## [v1.2.3] - 2026-05-12`。

## 发布步骤

1. 更新 `Directory.Build.props` 中的版本。
2. 在 `CHANGELOG.md` 添加匹配的 `vX.Y.Z` 小节。
3. 推送分支，等待 `ci` workflow 成功。
4. 创建并推送名为 `vX.Y.Z` 的 Git tag。
5. 等待 tag 触发的 `ci` run 发布安装包并创建 GitHub Release。

## 产物命名

- `CodexSwitch-vX.Y.Z-win-x64-setup.exe`
- `CodexSwitch-vX.Y.Z-linux-x64.AppImage`
- `CodexSwitch-vX.Y.Z-osx-arm64.dmg`
