---
title: 故障排查
description: 分层排查代理、provider、认证、路由和 release 问题。
icon: Wrench
---

先检查本地代理，再检查 provider 可见性，最后检查请求转换。这样可以避免把认证或路由问题误判为协议 bug。

## 代理排查

```powershell
curl http://127.0.0.1:12785/health
curl http://127.0.0.1:12785/v1/models
```

正常输出：

```text
/health returns 200 OK
/v1/models returns model data for the active provider
```

## Provider 排查

1. 确认 provider 已设为活动 provider。
2. 确认 API key 或 OAuth 账号存在。
3. 确认请求模型已映射到上游模型。
4. 确认 provider 协议与上游服务匹配。
5. 确认被禁用 provider 没有被预期处理流量。

## 受管理文件排查

1. 检查 `.codex` 或 `.claude` 受管理文件是否写入。
2. 检查同名 `.bak` 备份是否存在。
3. 停止代理，确认原文件恢复。
4. 如果使用 Codex App 插件，确认在应用代理配置前已启用 auth preservation。

## 发布排查

- 运行 `pwsh ./build/Validate-Changelog.ps1`。
- 确认 tag 是 `vX.Y.Z`。
- 确认 `Directory.Build.props` 版本与 changelog heading 一致。
- macOS 如被 Gatekeeper 阻止，检查签名与 notarization secrets。
