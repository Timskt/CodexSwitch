Provider 和协议变更通常会跨越多个层。

## 检查清单

- 新增或更新 provider template 数据。
- 确认协议选择和模型映射。
- 只在对应协议边界内更新 adapter 行为。
- 归一化用量，确保仪表盘和价格计算仍然正确。
- 保持受管理配置的恢复行为。
- 为路由、转换、用量和配置变更添加测试。

编辑前先阅读 [协议](/zh/docs/core-concepts/protocols) 和 [模型与路由](/zh/docs/core-concepts/models-and-routes)。
